- 文档版本：v2.1
- 修订日期：2026-03-19
- 修订依据：

## 1. 产品概述

### 1.1 产品定位
本模块为水务生产测试管理后台，覆盖从登录进入系统后所见的生产测试全链路界面，核心承载如下业务：
- 表具档案维护
- 仓库入库与设备状态管理
- 批次管理
- 批量测试与批量动作执行
- 批量测试历史快照与检验台数据绑定
- 包装管理
- 公告管理
- 账户、人员、角色管理

### 1.2 项目目标
- 将生产测试现场涉及的档案、仓库、批次、实时测试、历史追溯、包装等页面统一纳入一套 Web 管理后台。
- 明确前端界面中已经落地的字段、状态、按钮与弹窗边界，减少文档与界面不一致。
- 为后续接口、测试用例、验收和实施培训提供统一口径。

### 1.3 目标用户
- 平台管理员
- 企业管理员
- 生产操作员
- 仓库人员
- 包装人员
- 质检/测试人员

### 1.4 核心问题
- 原文档对页面字段覆盖不完整，尤其缺少批量测试、批量测试历史、检验台配置、包装三级层级等复杂界面。
- 原文档中部分页面命名、菜单口径与当前前端不一致。
- 原文档对部分弹窗字段描述过于抽象，无法支撑开发和测试落地。
- 原文档中存在超出当前界面范围的内容，容易导致范围扩散。

## 4. 角色与权限

### 4.1 角色口径
| 角色 | 说明 |
| --- | --- |
| 平台管理员 | 可查看并维护全部页面 |
| 企业管理员 | 可维护企业账户、人员、角色及业务数据 |
| 生产操作员 | 重点使用仓库、批量测试、包装相关页面 |

### 4.2 权限原则
- 登录后根据角色展示可见菜单。
- 账户、人员、角色三个页面属于系统管理范围。
- 批量测试、历史、包装等业务页面默认需要操作权限。
- 删除、停用、批量执行、保存历史等关键动作需二次确认并保留日志。

## 5. 核心业务流程

### 5.1 业务主链路
1. 登录系统。
2. 维护表具档案，形成基础型号、表具口径等档案口径。
3. 在仓库管理中录入或导入设备，形成设备在库台账。
4. 在批次管理中维护测试批次和备货单号分组。
5. 在批量测试页选择当前批次，将仓库设备逐台加入测试批次。
6. 在批量测试页执行批量动作、配置测试合格标准，并实时查看结果。
7. 执行“保存历史”后形成批量测试历史快照。
8. 在批量测试历史页查看历史详情，必要时绑定检验台 Excel 数据。
9. 在包装管理页按备货单和箱号完成扫码装箱与打印。

### 5.2 公共操作链路
- 任意页面可通过侧边栏切换模块。
- 页头支持全局搜索占位、系统状态显示、通知角标、用户信息展示。
- 平台公告条在所有登录后的业务页面顶部展示。
- 界面配置抽屉支持调整主题颜色和圆角配置。

## 6. 统一字段与状态口径

### 6.1 核心字段口径
 - 测试批号：格式参考 `{备货单号}-001`；同一备货单号下，后缀三位序号从 `001` 起递增
- 备货单号：格式参考 `2603-001`（示例类型）；用于作为测试批号前缀
- 水表表号：仓库页说明为 14 位唯一标识
- 箱号：包装页自动生成
- 表具名称 / 表具型号 / 表具口径：以表具档案口径为准
- 检验台记录：包含检验台名称、文件名、行数、上传时间

### 6.2 状态口径
| 类别 | 枚举 |
| --- | --- |
| 仓库库存状态 | `0-在库`、`1-已出库` |
| 仓库测试状态 | `0-待测`、`1-合格`、`2-不合格` |
| 批量测试判定状态 | 测试中、合格、不合格 |
| 批量测试阀门状态 | 关、开、故障 |
| 包装层级 | 备货单号层、箱号层、单表号层 |

### 6.3 详细字段参考
所有界面字段、下拉项、表格列、弹窗字段和按钮口径统一以 `字段界面.md` 为准。

### 6.4 必填字段标识与交互
- 必填字段在界面 `label` 右侧显示红色星号 `*`。
- 当用户点击“确定保存/提交”且必填为空时，触发前端校验提示（alert），星号保持可见。

补充说明：
- 除特别说明外，新增弹窗与编辑弹窗字段一致，编辑时默认回填原值。
- “系统生成 / 自动带出 / 只读”字段不允许人工直接修改。
- 本章为第一版补充口径，后续可在不改变页面结构的前提下继续细化校验规则、字典项和接口参数。

### 7.1 登录页
页面目标：
- 提供系统入口。

字段与交互：
| 字段 | 控件类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| 登录账号 | 输入框 | 是 | 支持账号输入 |
| 登录密码 | 密码框 | 是 | 密码隐藏展示 |
| 记住密码 | 复选框 | 否 | 勾选后下次自动回填账号信息 |
| 忘记密码 | 链接按钮 | 否 | 跳转找回或联系管理员 |
| 进入生产系统 | 主按钮 | - | 提交登录 |

新增弹窗字段：
- 无。

业务流程：
1. 用户输入登录账号和登录密码。
2. 点击“进入生产系统”。
3. 系统校验账号、密码、用户状态。
4. 校验通过后进入生产系统默认首页，并加载用户权限菜单。

边界情况：
- 登录账号为空时，提示“请输入登录账号”。
- 登录密码为空时，提示“请输入登录密码”。
- 账号或密码错误时，提示“账号或密码错误”。
- 账号被禁用时，禁止登录并提示联系管理员。
- 网络异常或服务不可用时，提示稍后重试。

### 7.2 公共框架页
页面目标：
- 提供统一导航、页头、公告和主题配置能力。

关键内容：
- 侧边栏菜单分三组：生产系统、公告模块、系统管理。
- 页头展示当前页面标题和面包屑。
- 页头提供全局搜索输入框、系统在线状态、通知角标、用户信息。
- 公告条滚动展示平台公告，支持点击“查看更多”进入公告管理页。
- 界面配置抽屉可调整主题颜色和圆角。

新增弹窗字段：
- 无固定“新增”弹窗。
- 界面配置抽屉字段建议包括：主题颜色（颜色选择器，非必填）、圆角大小（滑块/数字输入，非必填）、恢复默认（按钮）。

业务流程：
1. 用户登录后进入公共框架页。
2. 系统按角色加载侧边栏菜单和默认页。
3. 用户通过菜单切换功能页面，通过页头查看公告、通知和个人信息。
4. 用户可在界面配置抽屉中调整主题样式并即时生效。

边界情况：
- 当前用户无任何菜单权限时，展示无权限提示页。
- 公告为空时，公告条展示“暂无公告”。
- 当前页面标题与菜单未匹配时，面包屑按实际路由回退展示。
- 主题配置保存失败时，保留当前临时效果并提示用户重试。

### 7.3 公告管理页
页面目标：
- 查看平台公告列表，提供新增公告入口。

关键字段：
- 标题
- 优先级
- 内容摘要
- 日期
- 通知类型

新增/编辑公告弹窗字段：
| 字段 | 控件类型 | 是否必填 | 规则说明 |
| --- | --- | --- | --- |
| 标题 | 输入框 | 是 | 建议 1-50 个字符 |
| 优先级 | 下拉框 | 是 | 建议枚举：高 / 中 / 低 |
| 通知类型 | 下拉框 | 是 | 建议枚举：系统通知 / 测试通知 / 仓库通知 / 包装通知 / 其他 |
| 内容摘要 | 多行输入框 | 是 | 用于列表和顶部公告条展示，建议不超过 200 字 |
| 公告正文 | 富文本/多行输入框 | 是 | 具体通知内容 |
| 发布日期 | 日期时间选择器 | 是 | 默认当前时间 |

关键操作：
- 发布新公告
- 编辑公告
- 删除公告

业务流程：
1. 用户进入公告管理页查看现有公告列表。
2. 点击“新增公告”打开公告弹窗。
3. 填写标题、优先级、通知类型、摘要、正文和发布日期。
4. 点击“发布”后系统保存公告，并同步到业务页顶部公告条。

边界情况：
- 标题、摘要、正文任一为空时不允许发布。
- 发布日期为空或格式错误时不允许保存。
- 已发布公告被编辑后，应同步刷新公告条内容。
- 删除已在公告条展示的公告时，应同步从公告条移除。

### 7.4 表具档案页
页面目标：
- 维护测试、仓库、包装共用的表具档案基础信息。

查询条件：
- 关键字搜索（名称、型号、口径、备注）

列表字段：
- 表具名称
- 表具型号
- 表具口径
- 备注
- 创建时间

新增/编辑弹窗字段：
| 字段 | 控件类型 | 是否必填 | 规则说明 |
| --- | --- | --- | --- |
| 表具名称 | 输入框 | 是 | 必填；用于仓库、测试、包装统一引用 |
| 表具型号 | 输入框 | 是 | 必填；与名称共同标识档案型号 |
| 表具口径 | 输入框 | 是 | 必填；支持输入如 `15`、`20`、`DN15` 等口径描述 |
| 备注 | 多行输入框 | 否 | 建议不超过 200 字 |

关键规则：
- 协议、阀门、运营商采用固定枚举（不在表具档案维护）。
- 档案口径需供仓库、测试、包装引用。
- 表具名称、表具型号、表具口径均为输入框，且为必填项。

业务流程：
1. 用户进入表具档案页，通过列表查看已有档案。
2. 点击“新增”打开档案弹窗。
3. 输入表具名称、表具型号、表具口径，必要时填写备注。
4. 保存成功后，档案可在仓库管理、批量测试、包装管理中被引用。

边界情况：
- 表具名称、表具型号、表具口径任一为空时不允许保存。
- 表具名称 + 表具型号 + 表具口径完全重复时，不允许重复新增。
- 已被仓库或测试数据引用的档案，不允许物理删除；如需停用，建议转为停用状态或提示影响范围。
- 修改口径或型号时，如已被下游引用，应提示“修改后将影响已绑定数据展示口径”。

### 7.5 仓库管理页
页面目标：
- 承接设备入库、设备档案补充与出库状态管理。

页面结构：
- 顶部统计卡片
- 搜索与工具栏
- 设备列表
- 新增/编辑弹窗

关键字段：
- IMEI
- 水表表号
- 入库时间
- 正向累积流量
- 反向累积流量
- 备货单号
- 主副IP
- 表具名称
- 表具型号
- 口径
- 通讯协议
- 运营商
- 阀门状态
- 电池电压
- 装箱号
- 厂商代码
- ICCID
- CSQ / RSRP / RSRQ
- 库存状态
- 测试状态
- 出库时间
- 出库操作人员
- 备注
- 更新时间

新增/编辑弹窗字段：
| 字段 | 控件类型 | 是否必填 | 规则说明 |
| --- | --- | --- | --- |
| 表具名称 | 下拉框 | 是 | 来源于表具档案 |
| IMEI | 输入框 | 是 | 唯一校验 |
| 通讯协议 | 下拉框 | 是 | 固定枚举 |
| 运营商 | 下拉框 | 否 | 使用通信卡时必填 |
| 备货单号 | 下拉框 | 是 | 来源于批次管理的备货单号，用于设备归属分组 |
| 厂商代码 | 输入框 | 否 | 厂商追溯字段 |
| 备注信息 | 多行输入框 | 否 | 说明特殊情况 |

自动获取或系统生成字段：
- 正向累积流量、反向累积流量、阀门状态、电池电压、CSQ / RSRP / RSRQ 由设备上报或测试动作回写。
- 库存状态默认新增后为“在库”。
- 测试状态默认新增后为“待测”。
- 出库时间在执行出库动作后由系统生成。

关键操作：
- 扫码入库
- 新增入库
- 导入
- 批量出库
- 编辑
- 删除

关键规则：
- 水表表号为唯一标识。
- 自动获取字段和人工维护字段需分开处理。
- 设备状态与测试状态采用固定枚举。

业务流程：
1. 用户进入仓库管理页，查看库存统计和设备列表。
2. 点击“新增入库”或“扫码入库”打开弹窗。
3. 填写基础身份字段和档案字段，系统联动带出型号、口径等信息。
4. 备货单号为必填项，且从批次管理选择；未填写不允许保存。
5. 保存后设备进入“在库 / 待测”状态，可继续编辑、导入或纳入测试批次。
6. 需要出库时，用户执行批量出库，系统记录出库时间和出库人员。
7. 备货单号需要可模糊搜索

边界情况：
- 水表表号重复时不允许新增。
- IMEI 重复时不允许新增。
- 表具名称已选择但型号或口径未联动成功时，不允许保存。
- 导入文件存在部分无效数据时，应支持失败行提示，不影响有效行导入。
- 已进入测试批次或已生成历史快照的设备，不允许删除。
- 已出库设备再次执行出库时，应提示“设备已出库”。
- 通讯协议需要通信卡时，ICCID 或运营商为空应提示补充。
- 导入的时候需要校验表里面的IMEI是否存在，否则拒绝导入

### 7.6 批次管理页
页面目标：
- 按备货单号组织测试批次，并提供批次维护能力。

页面结构：
- 搜索区

| --- | --- | --- | --- |
| 备货单号 | 输入框 | 是 | 用户输入，需校验唯一；保存后仅创建备货单号分组（不生成测试批次） |
| 备注 | 多行输入框 | 否 | 建议不超过 200 字 |

编辑备货单号弹窗字段：
| 字段 | 控件类型 | 是否必填 | 规则说明 |
| --- | --- | --- | --- |
| 当前备货单号 | 只读输入框 | 是 | 展示当前值 |
| 新备货单号 | 输入框 | 是 | 保存后同步更新该组全部测试批号前缀：`{旧备货单号}-序号` -> `{新备货单号}-序号`（保留后缀序号） |

关键规则：
- 列表按备货单号分组。
- 修改备货单号时，需要同步更新该组全部测试批号前缀（保留后缀序号与递增规则）。
- 待测数量和测试进度由数量字段推导。

业务流程：
1. 用户进入批次管理页，按备货单号查看已有批次。
2. 点击“新增备货单号”，只填写备货单号和备注。
3. 点击保存后，该备货单号分组写入列表并自动统计（当前暂无测试批次，合计为 0）。
4. 在备货单号列表行的操作列，可继续“新增批次号”，系统自动按后缀递增生成下一条测试批号（无需用户输入）；并跳转到批量测试页，默认选中刚创建的测试批号。
5. 用户可在批量测试页按测试批号继续加表和执行测试。
6. 如备货单号录入错误，可通过“编辑备货单号”统一调整该组数据。

边界情况：
- 备货单号重复时不允许保存。
- 删除备货单号：同时删除该备货单号下的全部测试批次。
- 删除测试批号：仅删除该测试批次。
- 编辑备货单号时，应同步更新该组全部测试批号前缀；若新单号下导致批号冲突，则应失败并提示用户重试。
- 备货单号不存在可用仓库设备时，应提示“当前备货单无可用设备”。
- 一个备货单号下允许存在多个测试批次，但每个测试批号必须唯一。
- 已产生测试历史的批次若修改备货单号，应二次确认，避免历史追溯混乱。
- 系统统计数量异常时，应支持重新统计，不允许手工改写系统字段。

- 已在当前批次内的设备重复添加时，应提示“设备已存在于当前批次”。
- 批量动作执行中再次点击同一动作时，应限制重复提交。
- 批量动作出现部分成功、部分失败时，应保留逐台执行结果。
- 测试标准参数未配置完整时，不允许系统自动判定合格。
- 保存历史时，如仍有待测设备，应二次确认是否继续保存。
- 同一批次重复保存历史时，应提示是否生成新的历史快照版本。
-设备检验合格标准：
  用户可以设置 ，默认是
  电池电压大于3.6V，NB信号强度大于10，4G大于13，无磁大于130
- 用户可以手动保存到历史、 可以手动合格 
- 手工合格之后，代码不判断

- 一条检验台数据被重复绑定到多个平台表号时，应禁止保存。
- 存在未匹配数据时，应允许部分保存，但需提示剩余未匹配数量。
- 已绑定的检验台记录如被删除，应先解除绑定或禁止删除。

| 箱号 | 只读输入框 | 是 | 按固定规则自动生成 |
| 测试批号条码信息 | 只读区域 | 是 | 系统自动生成条码内容 |

| 表具口径 | 只读输入框 | 是 | 自动获取 |

| 备注 | 多行输入框 | 否 | 包装异常说明 |

关键规则：
- 列表采用备货单号层、箱号层、单表号层三级展示。
- 箱号自动生成。
- 条码信息用于打印与核对。
- 检验员、包装时间为系统生成字段。
- 当点击“下一箱”时，系统将按顺序执行：保存当前包装记录 -> 打印当前箱标签 -> 清空已扫码表号并重置进度 -> 自动切换到下一箱（箱号自动累积+1，预览条码同步更新）。

业务流程：
1. 用户进入包装管理页，按备货单号或箱号定位待包装数据。
2. 点击“新增包装记录”，系统生成箱号和条码信息。
3. 用户连续扫码水表表号，系统自动校验设备是否已测试合格且未包装。
4. 扫码完成后（达到当前箱容量，默认 10 个水表表号）可选择：点击“保存记录”保存当前箱；或点击“下一箱”触发“保存 -> 打印 -> 清空已扫码表号并重置进度 -> 自动切换到下一箱（箱号自动累积+1，预览条码同步更新）”。
5. 用户可在已保存的箱号上点击“打印标签”重新打印条码或包装清单。

边界情况：
- 未通过测试的设备不允许包装。
- 已包装设备再次扫码时，应提示“设备已包装”。
- 未达到当前箱容量（默认 10 个水表表号）时，不允许保存或进入下一箱。
- 扫码表号不存在时，应提示“未找到对应设备”。
- 箱号生成冲突时，系统应自动重试生成，避免重复。
- 打印失败不影响包装保存，但需提示用户重新打印。

- 企业编码重复时不允许保存。
- 登录账号重复时不允许保存。
- 新增时初始密码为空不允许保存。
- 已停用账户不允许登录系统。
- 若账户下仍存在有效人员，不建议直接删除，应优先停用。

#### 7.10.2 人员管理
关键字段：
- 登录账号
- 人员姓名
- 所属企业
- 所属角色
- 最后登录
- 状态

新增/编辑弹窗字段：
| 字段 | 控件类型 | 是否必填 | 规则说明 |
| --- | --- | --- | --- |
| 登录账号 | 输入框 | 是 | 唯一登录名 |
| 真实姓名 | 输入框 | 是 | 页面展示名称 |
| 所属企业 | 下拉框 | 是 | 来源于账户管理 |
| 所属角色 | 下拉框 | 是 | 来源于角色管理 |
| 初始密码 | 密码框 | 新增时必填 | 编辑时为空表示不修改 |
| 状态 | 开关 | 是 | 启用 / 停用 |

业务流程：
1. 管理员进入人员管理页查看人员列表。
2. 点击“新增人员”，填写账号、姓名、所属企业、所属角色。
3. 保存成功后，人员即可按角色权限登录系统。
4. 如组织或职责变化，可编辑所属企业、角色和状态。

边界情况：
- 登录账号重复时不允许保存。
- 所属企业为空时不允许保存。
- 所属角色为空时不允许保存。
- 引用已停用角色时，应提示重新选择角色。
- 当前登录用户不可删除自身账号。

#### 7.10.3 角色管理
关键字段：
- 权限角色
- 说明
- 功能权限

新增/编辑弹窗字段：
| 字段 | 控件类型 | 是否必填 | 规则说明 |
| --- | --- | --- | --- |
| 角色名称 | 输入框 | 是 | 唯一角色标识名称 |
| 角色描述 | 多行输入框 | 否 | 角色说明 |
| 功能权限 | 树形复选框/穿梭框 | 是 | 勾选可访问页面和操作权限 |

业务流程：
1. 管理员进入角色管理页，查看现有角色及权限范围。
2. 点击“新增角色”，填写角色名称、描述并配置功能权限。
3. 保存后，角色可在人员管理页中被引用。
4. 编辑角色权限后，新权限对已绑定人员即时生效或重新登录生效。

边界情况：
- 角色名称重复时不允许保存。
- 未勾选任何功能权限时不允许保存。
- 已被人员引用的角色删除前应提示影响范围。
- 系统内置角色如存在，不允许删除，仅允许查看或部分编辑。

关键规则：
- 人员管理的所属企业和所属角色来自已有列表。
- 角色权限当前以前端展示为准，典型值如 `all`、`testing`、`packaging`。
